# Moment Card KPI — UniCard Dashboard

Internal KPI and card-operations platform for a partner-distributed VISA card program. Two admin roles, one codebase, installable as a PWA.

- **Bank Admin** — oversees every partner organization, card batch, KPI config, reward payout, notification rule, delivery log, and system announcement across the platform.
- **Organization Admin** — runs a single organization: sellers, card inventory, KPI conversion, UCOIN withdrawals, and direct messaging to sellers.

The live SPA targets desktop 1920×1080, adapts to mobile below 768 px, and can be installed as a Progressive Web App on Chrome/Edge/Android + iOS Safari.

## Tech stack

- React 18 + TypeScript + Vite 6
- react-router 7 (`createBrowserRouter`)
- Tailwind CSS 4 with `@theme inline` tokens (inline styles are the primary mechanism — Tailwind is used sparingly under `components/ui/`)
- @radix-ui + shadcn/ui primitives
- recharts · motion · react-hook-form · lucide-react · date-fns
- vite-plugin-pwa 1.2 + workbox-window 7 (auto-update service worker, installable manifest)

## Running the code

This project uses **pnpm**, not npm. The `package.json` declares `react` / `react-dom` as optional `peerDependencies` (a pnpm convention); running `npm install` silently drops React from `node_modules` and the build fails.

```bash
pnpm install
pnpm dev       # Vite dev server on :5173
pnpm build     # production build → dist/ (includes sw.js + manifest.webmanifest)
pnpm exec vite preview   # preview the built bundle on :4173 (use this to test PWA install)
```

If `package-lock.json` appears after a stray `npm install`, delete it — `pnpm-lock.yaml` is the source of truth.

## Progressive Web App

Shipping as an installable PWA:

- Service worker generated by workbox, strategy `autoUpdate` — new SW takes over on the next navigation
- Manifest: `Moment KPI`, `display: standalone`, brand-blue theme `#2563EB`, 192/512 any + 512 maskable icons
- `useInstallPrompt()` surfaces a native install row in Mobile Settings → "Приложение" with 4 states (installed / can-install / iOS help sheet / waiting)
- 20 iOS launch images in `public/splash/` (10 iPhone sizes × light/dark) wired into `index.html`
- Runtime caching: NetworkFirst for pages (3 s timeout), StaleWhileRevalidate for scripts, CacheFirst for images + fonts
